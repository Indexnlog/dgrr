## 상세

### ✅ 수업 관리
- Event CRUD, 출석/지각/불참, 수업 종료
- 일정 캘린더 통합 (매치+수업)

### ✅ 투표 시스템
- 투표 생성/투표하기/종료
- 홈 탭 활성 투표 배너
- Optimistic UI, 옵션별 로딩 표시
- **미완료**: 투표 종료 시 Event 자동 생성

### ✅ 매치 고도화
- D-day 배지, 경기 생성, 라인업 편집
- 빠른 기록 (골/교체/상대팀 골)
- 상대팀 수정, currentFieldPlayers
- 공 가져오기 ("저도 들고가요" 자원 방식)
- 참석/불참 Optimistic UI

### ✅ 푸시 알림 (클라이언트)
- FCM 권한 요청, 토큰 Firestore 저장
- 백그라운드 메시지 핸들러
- **미완료**: Nudge Cloud Function (`docs/FCM_NUDGE_연동가이드.md`)

### ✅ 구장 예약
- 예약 공지, 성공/실패 보고, Ground 관리

### ✅ 공지 시스템
- 최근 공지 1개 표시, 공지 작성 UI
- 공지 목록/상세 페이지
- 공지 작성 템플릿(20일 등록 공지 초안) 자동 채움
- 공지 목록 페이지네이션 2차 적용(고정 우선 정렬, 중복 방지, 끝 상태 표시)
- 공지 제목/내용 검색 1차 반영
- 작성/수정 권한 UX(권한 없음 안내 화면) 반영

### ✅ 리스트 성능 (1차)
- 투표 목록 증분 로드(페이지 limit 증가 방식)
- 경기 목록 증분 로드(페이지 limit 증가 방식)

### 🔄 에러 처리 공통화 (1차)
- Firebase 예외 -> AppException 매핑 유틸 추가 (`firebase_exception_mapper.dart`)
- 게시글/투표/경기 데이터소스에 예외 매핑 1차 적용
- 목록/상세/작성 화면에서 ErrorHandler 기반 사용자 메시지 적용

### ✅ 에러 처리 공통화 (2차)
- `poll_create`, `poll_detail`, `match_detail`, `class_detail`에 ErrorHandler 메시지 적용
- 경기/라운드 주요 액션(시작/종료/추가) 실패 시 사용자 안내 SnackBar 처리
- 수업 상세 출석 투표 실패/팀 미선택 시 로딩 상태 복구 보강

### ✅ 에러 처리 공통화 (3차)
- 예약 화면(`reservation_notice_list/detail/create`) ErrorHandler 기반 메시지 적용
- 알림/팀설정 화면 에러 메시지 공통화
- 예약 상세 로딩 실패 UI를 ErrorRetryView로 통일

### ✅ 린트 정리 스프린트 (1차)
- `use_build_context_synchronously` 주요 경고 정리(알림/팀설정/투표생성)
- `curly_braces_in_flow_control_structures` 주요 경고 정리(홈/투표상세)
- `flutter analyze` 이슈 수: 66 -> 57 감소

### ✅ 회비 관리
- FeeManagementPage, 회비 설정
- MY 회비 납부 현황 페이지 (`/my/fee-status`)
- 회비 관리 → 등록 투표 보기/공지 초안 작성 연결

## UI/UX (최근 반영)

- 상단 프로필 + 알람(벨) 창
- 색조합: 그린 배너, 블루 선택일
- 달력: 이벤트 날짜 그린 서클, 선택일 블루
- Optimistic UI: 투표/참석 탭 즉시 반영
- 옵션별 로딩: 투표 저장 중 해당 옵션만 스피너
- 공 가져오기: "저도 들고가요" 칩 (홈/경기 상세)
