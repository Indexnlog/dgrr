# 운영 Runbook (출시 후)

> 목적: 장애를 "빨리 감지하고, 안전하게 완화하고, 재발을 막는" 표준 절차

---

## 1. 운영 원칙

- 사용자 영향 최소화가 최우선
- 원인 분석보다 **즉시 완화(Kill Switch/권한 차단)** 먼저
- 모든 조치는 `audit_logs`에 남기고, 종료 후 회고 문서화

---

## 2. 장애 등급

| 등급 | 기준 | 목표 대응 시간 |
|------|------|----------------|
| P0 | 로그인 불가, 데이터 손상 위험, 결제/권한 오작동 | 15분 이내 완화 |
| P1 | 핵심 플로우 일부 실패율 급등 | 1시간 이내 완화 |
| P2 | 비핵심 기능 문제/우회 가능 | 24시간 이내 수정 |

---

## 3. 공통 대응 절차 (P0/P1)

1. **감지**
   - Crashlytics 신규 치명 에러
   - `ops_metrics_daily` 실패율 급등
2. **완화**
   - 위험 기능 off (Feature Flag/Kill Switch)
   - 필요 시 관련 callable 임시 차단
3. **확인**
   - 실패율 재하락 확인
   - 신규 치명 에러 발생 중단 확인
4. **복구**
   - 안전 패치 배포
   - 점진적으로 기능 재활성화
5. **회고**
   - 원인/영향/재발 방지 액션 기록

---

## 4. 시나리오별 즉시 액션

### 4-1. 회비 확정 실패 급증
- 임시 조치: 회비확정 UI 버튼 비활성화
- 확인 항목:
  - `updateRegistrationPaymentStatus` 실패 코드 분포
  - App Check 실패 비율(`failed-precondition`)
  - 권한 오류 비율(`permission-denied`)

### 4-2. 예약 성공/실패 보고 실패 급증
- 임시 조치: 예약 결과 보고 버튼 비활성화
- 확인 항목:
  - `reportReservationResult` 오류 코드
  - notice 문서 구조/slots 데이터 일관성

### 4-3. 미납 알림 발송 이상(중복/폭주)
- 임시 조치: Nudge callable 차단 또는 rate limit 강화
- 확인 항목:
  - requestId 중복 여부
  - `sendNudgeToUnpaid` 호출량/실패량

### 4-4. App Check 오탐(정상 사용자 차단)
- 임시 조치: 해당 서비스 Enforcement 일시 완화
- 확인 항목:
  - 플랫폼별 토큰 발급 상태
  - 웹 site key 설정 누락 여부

---

## 5. 배포/롤백 절차

### 배포 전
- [ ] `npm run build` (functions)
- [ ] `npm run test:rules` 통과
- [ ] `firebase deploy --only functions,firestore:rules`

### 롤백 기준
- P0 발생 + 15분 내 완화 실패
- 신규 릴리스 후 실패율 3배 이상 급등

### 롤백 실행
1. 기능 토글 OFF (즉시)
2. 이전 안정 버전 함수 재배포
3. Rules 변경이 원인일 경우 직전 버전 복원
4. 영향 데이터 수동 보정 계획 수립

---

## 6. 운영 일일 점검 (5분)

- [ ] 오늘 치명 크래시 0건 확인
- [ ] 핵심 함수 실패율 임계치 이내 확인
- [ ] `audit_logs`에 이상 권한 변경 없는지 확인
- [ ] `member_tokens` active 비율 급감 여부 확인

---

## 7. 주간 점검 (30분)

- [ ] 실패율 상위 3개 액션 원인 분류
- [ ] rate limit 초과 패턴 검토(봇/오사용 징후)
- [ ] App Check 거부율 추이 점검
- [ ] 백업 스냅샷 정상 생성 확인
