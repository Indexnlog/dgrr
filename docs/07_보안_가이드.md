# 보안 가이드

> 2026-03 기준 운영 규칙. 기능보다 보안/운영 안정성 우선.

---

## 1. 현재 적용 상태 (핵심)

- Firestore Rules: `teams/{teamId}` 멀티테넌트 격리 적용
- 민감 액션: 클라이언트 직접 write 금지 + Cloud Functions 경유
  - 회비 납부 상태 변경
  - 예약 성공/실패 보고
  - 미납자 Nudge 발송
- FCM 토큰: `member_tokens` 분리 저장(본인/서버만 접근)
- App Check: 앱 초기화 코드 반영(웹/모바일)
- Functions 보호: App Check 검증 + idempotency(requestId) + rate limit(분당 제한)

---

## 2. 역할 권한 매트릭스 (요약)

| 액션 | 일반 | 운영진/총무 | 서버(Function) |
|------|------|-------------|----------------|
| 본인 등록 투표(status != paid) | ✅ | ✅ | ✅ |
| 회비 `paid` 확정/해제 | ❌ | ❌(클라 직접 금지) | ✅ |
| 예약 성공/실패(`slots/status`) | ❌(클라 직접 금지) | ❌(클라 직접 금지) | ✅ |
| 알림 생성/삭제 | ❌ | ❌ | ✅ |
| 알림 읽음(`readBy`) | ✅ | ✅ | ✅ |
| FCM 토큰 읽기 | 본인만 ✅ | 타인 ❌ | ✅ |

> 운영진/총무 권한이 있어도 민감 액션은 서버 함수 호출로만 처리한다.

## 3. Firestore Rules 검증 (스모크 테스트)

- 위치: `functions/src/firestore_rules_smoke_test.ts`
- 실행:

```bash
# 1) Firestore Emulator 실행 (프로젝트 루트)
firebase emulators:start --only firestore

# 2) Rules 테스트 실행 (functions 폴더)
npm run test:rules
```

- 검증 항목:
  - 일반 멤버 `paid` 변경 차단
  - 총무 `paid` 변경 허용
  - 알림 생성 차단 / 읽음 처리만 허용
  - 예약 slots 직접 수정 차단
  - 본인 members 문서 권한 상승(role 변경) 차단
  - `member_tokens` 본인만 읽기 허용

## 4. App Check 적용 가이드 (최소)

### 4-1. 코드
- `lib/main.dart`에서 `FirebaseAppCheck.instance.activate()` 호출
- emulator 모드(`USE_FIREBASE_EMULATOR=true`)는 App Check 생략

### 4-2. 콘솔 설정
- Android: Play Integrity
- iOS: App Attest(+ DeviceCheck fallback)
- Web: reCAPTCHA v3/Enterprise site key 발급

### 4-3. 운영 전환 순서
1. 모니터링 모드(Enforcement off)
2. 정상 트래픽/실패율 확인
3. Firestore/Functions/Storage 순서로 Enforcement on

---

## 5. FCM 토큰 운영 정책

- 저장 경로: `teams/{teamId}/member_tokens/{uid}`
- 필드: `fcmToken`, `active`, `updatedAt`
- 로그아웃 시:
  - 디바이스 `deleteToken()`
  - 서버 문서 `active=false`, `fcmToken=null`
- 마이그레이션 기간:
  - Functions에서 `members.fcmToken` 폴백 조회 허용
  - 안정화 후 폴백 제거

---

## 6. 운영성 데이터(서버 전용)

- `teams/{teamId}/audit_logs`
  - 서버 append-only
  - 운영진/총무 read-only
  - 필드: `actorUid`, `action`, `targetType`, `targetId`, `before`, `after`, `meta`, `createdAt`
- `teams/{teamId}/ops_metrics_daily/{yyyy-mm-dd}`
  - 서버 집계 전용
  - 실패율 카운터 저장 (`failures.{action}.total`, `failures.{action}.codes.{errorCode}`)
