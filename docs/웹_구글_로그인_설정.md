# 웹 구글 로그인 설정 (Client ID)

웹에서 `flutter run -d chrome` 실행 시 "ClientID not set" 오류가 나면 아래 순서대로 설정하세요.

---

## 1. Web Client ID 발급

1. **[Google Cloud Console](https://console.cloud.google.com/apis/credentials?project=dgrr-app)** 접속
2. 프로젝트 `dgrr-app` 선택
3. **"사용자 인증 정보"** → **"OAuth 2.0 클라이언트 ID"** 목록 확인
4. **"웹 애플리케이션"** 타입이 없으면:
   - **"+ 사용자 인증 정보 만들기"** → **"OAuth 클라이언트 ID"**
   - 애플리케이션 유형: **웹 애플리케이션**
   - 이름: `dgrr-app-web` (또는 원하는 이름)
   - **승인된 JavaScript 원본**에 추가:
     - `http://localhost`
     - `http://localhost:포트번호` (예: `http://localhost:7357`)
     - 배포 시: `https://yourdomain.com`
5. **만들기** 클릭 후 **클라이언트 ID** 복사 (형식: `27853679688-xxxxx.apps.googleusercontent.com`)

---

## 2. web/index.html 수정

`web/index.html`에서 아래 줄을 찾아 `content` 값을 실제 클라이언트 ID로 교체하세요.

```html
<meta name="google-signin-client_id" content="여기_클라이언트_ID_붙여넣기">
```

예시:
```html
<meta name="google-signin-client_id" content="27853679688-abc123xyz.apps.googleusercontent.com">
```

---

## 3. 다시 실행

```bash
flutter run -d chrome
```

---

## 참고

- 로컬 테스트 시 `http://localhost`가 승인된 원본에 있어야 함
