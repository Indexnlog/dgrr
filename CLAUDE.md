# dgrr

## Purpose
- Personal product and app experimentation repo.
- Keep this repository documented enough for navigation, but lighter than work-linked project standards.

## Repo Map
- `lib/`: main Flutter application code.
- `docs/`: supporting product and implementation docs.
- `functions/`: backend or serverless logic.
- `assets/`: static assets.
- `test/`: automated tests.

## Rules
- Treat this as a personal repo unless its operating context changes.
- Keep product docs, schema docs, and app code aligned when core flows change.
- Avoid mixing work-linked operating rules into this repo.

## Common Commands
```bash
flutter run
flutter test
```
